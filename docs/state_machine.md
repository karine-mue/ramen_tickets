## 1. 券売機取引状態遷移

```mermaid
stateDiagram-v2
    [*] --> S0_待機中

    S0_待機中 --> S1_商品選択中: 画面タッチ
    S1_商品選択中 --> S2_注文構成中: ラーメン/サイド選択
    S2_注文構成中 --> S3_注文確認中: カート確認
    S3_注文確認中 --> S2_注文構成中: 修正
    S3_注文確認中 --> S4_決済前チェック中: 注文確定

    S4_決済前チェック中 --> E1_販売不可: 釣銭不足/券紙切れ/通信不可
    S4_決済前チェック中 --> S5_入金受付中: チェックOK

    S5_入金受付中 --> S2_注文構成中: 未入金キャンセル
    S5_入金受付中 --> S6_入金済み: 必要額到達

    S6_入金済み --> S7_ローカル取引保存: transaction永続化
    S7_ローカル取引保存 --> S8_注文受付サーバ送信: 冪等キー付き送信

    S8_注文受付サーバ送信 --> S9_中央採番済み: order_id/display_no取得
    S8_注文受付サーバ送信 --> E2_送信結果不明: 通信断/応答なし

    S9_中央採番済み --> S10_厨房キュー登録済み: server側登録完了
    S10_厨房キュー登録済み --> S11_発券処理中: 食券/控え印刷

    S11_発券処理中 --> E3_発券失敗: プリンタ故障/紙詰まり
    S11_発券処理中 --> S12_釣銭返却中: 発券成功

    S12_釣銭返却中 --> E4_返金失敗: 釣銭不足/排出失敗
    S12_釣銭返却中 --> S13_取引完了: 釣銭返却成功/釣銭なし

    S13_取引完了 --> S0_待機中: 完了表示終了

    E1_販売不可 --> S0_待機中: 注文取消
    E2_送信結果不明 --> R1_再送待ち
    R1_再送待ち --> S8_注文受付サーバ送信: 同一transaction_idで再送
    R1_再送待ち --> M1_店員対応待ち: retry上限

    E3_発券失敗 --> M1_店員対応待ち
    E4_返金失敗 --> M1_店員対応待ち
```

## 7. 状態設計上の要点

```text
- 画面状態と取引状態を分離する
- 決済済み状態は永続化する
- 厨房送信待ち・再送待ちは未来実行義務として扱う
- 保守モード移行前に未完了取引を必ず確認する
- 起動時に未完了取引を必ずスキャンする
- トッピングは必ずMealUnitに紐付ける
- 中央採番点でdisplay_noを直列化する
- transaction_idを冪等キーとして二重注文を防止する
